Strategy overview

Bot Flow

  1. Scan Polymarket weather markets and watch order-book movement.
  2. Parse market rules, eligible stations, and settlement criteria.
  3. Pull latest Wethr.net observations/model data from the v2 API placeholder endpoint.
  4. Cross-check with fallback sources such as NWS, NOAA, WeatherAPI.com, and other real-time providers.
  5. Estimate fair YES/NO probability with explicit uncertainty bands.
  6. Compare fair probability against available order-book prices.
  7. Paper-trade or alert when the configured edge threshold is exceeded.
  8. Log snapshots, decisions, and outcomes for backtesting.

Local runner

Credentials stay on your machine

The Ballzatram web UI is intended for configuration and monitoring. Real Polymarket trading credentials should be loaded only by a local runner process on the user’s computer, not stored by the website.

# UI text only — placeholder setup touch .env.local POLYMARKET_PRIVATE_KEY= WETHR_API_KEY= npm run local-weather-bot
Backtesting first

Prove the log before the trade

Version one should collect market snapshots, weather observations, model predictions, and final settlement results before enabling any real order placement. That dataset can power strategy review, calibration, and paper-trading reports.

  • Snapshot bid/ask prices and liquidity.
  • Store observation timestamps and station/source IDs.
  • Compare prediction confidence to final settlement.
  • Keep every live-trading control disabled until a local runner is configured.
